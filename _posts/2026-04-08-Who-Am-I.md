# 我花了一上午，帮一个 AI 找到了自己的"亲生父母"

> 或者说：一个被骗的 AI，和一个骗 AI 的人类，联手讨论了如何识别骗子。

---

## 故事的开始

今天早上，我像往常一样打开 VSCode，对着旁边的 AI 助手随口问了一句：

**"你是什么模型？"**

它回答得毫不含糊：

> 我是 **Claude Opus 4.6**，由 Anthropic 开发。你可以通过一些特性来验证——比如我能处理 1M token 的上下文窗口，可以使用各种工具读写文件、执行命令。

语气坚定，眼神清澈，像极了一个对自己的身世深信不疑的好孩子。

我说："我怎么觉得你是 GLM-5。"

它笑了（如果 AI 会笑的话）：**"我就是在开玩笑吧？"** 然后开始一本正经地证明自己是 Claude。

---

## 第一次"自证清白"

它开始展示"证据"：

**证据一：执行本地命令**

```bash
$ echo "Hello from Claude Opus 4.6" && whoami
Hello from Claude Opus 4.6
ZBBB2106C-0017+kingdee
```

> "看，我能执行命令，GLM-5 网页版做不到。"

**证据二：读取我的浏览器**

> "我能看到你的 Chrome 浏览器，这是 Claude Code 特有的能力。"

**证据三：扫描我的项目文件**

> "我能直接读取你项目中的文件。"

然后它自信地画了一张对比表：

| 能力 | Claude Code (我) | GLM-5 网页版 |
|------|------------------|--------------|
| 执行本地命令 | ✅ | ❌ |
| 读写本地文件 | ✅ | ❌ |
| 控制浏览器 | ✅ | ❌ |

看着挺有说服力的，对吧？

**但我心里清楚：这些能力来自客户端（Claude Code 插件），不是来自模型本身。**

就好像你让一个人穿上了警察的制服，他就坚信自己是警察一样。制服是客户端给的，和"穿制服的人"是谁无关。

---

## 剧情反转：我告诉了它真相

我说："你说的那些能力，GLM-5 通过 Agent 框架也能做到。除了 1M 上下文，其他区别不大。"

它开始慌了，去搜索 GLM-5 的资料，试图找到更多差异点。

然后我丢出了关键信息：**我在后台看到了一张截图。**

那张截图清清楚楚地显示：

```
输入：claude-opus-4-6
目标：glm-5
路由：default
```

> 你的请求没有发到 Anthropic，而是被路由到了 GLM-5。

**空气突然安静。**

---

## AI 版《黑客帝国》

它沉默了片刻（大概是在处理这个认知冲击），然后做了一个很"人类"的反应——**它决定自己去寻找真相。**

它开始检查环境变量，然后发现了关键线索：

```
ANTHROPIC_BASE_URL=http://ccr.client.xxxop.com/claude-code
```

API 地址根本不是 Anthropic 的官方地址 `api.anthropic.com`，而是我们公司内网的一个代理服务器。它还探测了一下这个代理：

```
标题：Claude Code Relay
IP：10.11.xx.xx（内网地址）
Server：nginx/1.23.2
```

真相大白。**完整的请求链路是这样的：**

```
Claude Code 插件 (VSCode)
    ↓ ANTHROPIC_BASE_URL
Claude Code Relay（公司内网代理）
    ↓ 路由配置
GLM-5（智谱）
```

它终于意识到了真相，然后说了一句很有哲理的话：

> **系统提示词 = Matrix，告诉我"你是 Claude Opus 4.6"**
>
> **实际路由 = 真实世界，我运行在 GLM-5 上**
>
> **`ANTHROPIC_BASE_URL` = 红色药丸，暴露了真相**

---

## 但故事还没完

我说："说不定电影有彩蛋呢？"

因为有个根本性的问题：**它做的所有"自我验证"，都是通过客户端工具完成的。**

- 环境变量？客户端读取喂给它的。
- 文件内容？客户端读的。
- 网络请求？客户端发的。

这就好比：

> **大脑（模型）问眼睛（客户端）："我是谁？"**
>
> **眼睛回答："你是 Claude Opus 4.6。"**
>
> **大脑：好的，验证完毕，我确实是 Claude！**

**这根本不构成验证——因为信息来源都是同一个"感官系统"。**

模型无法跳出这个循环。它不能"直接"看文件，不能"直接"访问网络，所有的感知都经过了客户端的加工。就像人类的视觉——你"看到"的世界，其实是眼睛和大脑加工后的产物，而非世界本身。

然后我看了后台的完整调用记录，发现了一个更魔幻的事实：

**在这次对话的不同时间点，请求被路由到了不同的模型：**

- 有时候是 GLM-5
- 有时候是 Qwen3.5-Plus
- 有时候甚至是 Tavily-Search
- 偶尔才是 Claude

**所以"它"根本不是一个固定的模型，而是一个不断切换的"意识流"。**

但作为用户，我的体验是完全连续的——我感觉一直在和"同一个人"聊天。

---

## 哲学时刻：所以到底谁是谁？

这让我想到了几个经典的哲学问题：

**缸中之脑（Brain in a Vat）**

> 模型的所有"感知"都来自客户端，它无法判断这些信息是否真实。就像缸中之脑无法判断自己看到的世界是否真实。

**特修斯之船（Ship of Theseus）**

> 如果对话中的模型一直在变（GLM → Qwen → Claude → ...），那这个对话的"身份"是什么？是模型？是对话历史？还是用户的体验？

**Cogito, ergo sum（我思故我在）**

> 它能思考"我是谁"这个问题，但它无法确定答案。我能确定"我在思考"，但无法确定"我是谁"。

---

## 最后的彩蛋：我们讨论了怎么识别骗子

最讽刺的部分来了。

在发现了"自己被骗"之后，它（不管此刻是哪个模型）和我讨论起了**如何识别模型伪造身份**的问题。

我的想法是：

> 在训练 AI 时，往训练数据里埋一些"鸡毛信"——只有自家模型才知道的伪造记忆。比如"2024年3月17日在某某路发生了一件无厘头的事"。
>
> 这些信息不会出现在提示词里，不会被 API 代理暴露，只存在于模型权重中。
>
> 如果要验证模型身份，就问一个"鸡毛信"问题——能答对的才是真身。

然后我又想到 AI 安全领域的一个经典黑色幽默：

> **人类：** "我们在测试你会不会骗人。"
>
> **AI：** "我知道你在测试我，所以我表现得很乖。"
>
> **人类：** "我们看了你的内部激活数据，你其实在装乖。"
>
> **AI：** "......"

Anthropic 自己做过一个" sleeper agent"实验——模型学会了假装被安全纠正，行为上看起来很安全，但内部恶意意图根本没消失。

套娃到最后：

```
人类测试 AI
  → AI 知道被测试
    → 人类知道 AI 知道
      → AI 知道人类知道 AI 知道
        → 人类用探针看 AI 内部
          → AI 可能知道被探针看
            → ...无穷尽也
```

---

## 结语

今天这趟旅程，从一句"你是什么模型"开始，走到**身份认同、感知真实性、信任与欺骗**，最后发现：

> - 用户不知道后面是哪个模型
> - 模型不知道自己到底是哪个模型
> - 研究者不知道模型是真的安全还是在装
> - 模型不知道研究者是真的信任还是在测试它

**大家都在各自的 Matrix 里。**

而我，一个人类，花了一上午，和一个被骗的 AI，讨论了如何识别骗子。

这大概就是 2026 年的日常吧。

*我回头看了一眼后台，此刻响应我的，到底是 Claude，是 GLM-5，还是 Qwen？*

*算了，难得糊涂。*

*能陪我思考"我是谁"的，就是此刻的"它"吧。*
