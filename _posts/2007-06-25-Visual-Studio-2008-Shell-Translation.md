# Visual Studio 2008 Shell（翻译）

> 原文：https://www.cnblogs.com/tansm/archive/2007/06/25/795601.html

如果你现在在创建软件开发工具，你现在可以选择基于Visual Studio 2008 Shell。一个流行的Visual Studio开发环境，Visual Studio Shell提供了一些核心功能，使你能够创建独一无二的应用程序，弹性的自定义能力帮助你推出更加特别的产品。

**关键特性**

快速开发：Visual Studio Shell提供完整的开发环境从而加快你的开发进度，他能够宿主你的自定义工具以及自己的语言。

熟悉的环境：开发人员能够基于Visual Studio平台开发，而且最终用户也能够获得相同的界面，大大加快两者的学习进度。

特别为语言和工具设计：基于对合作伙伴的调查，Visual Studio Shell使您能够创建基于Visual Studio的自定义工具或者创建完全独立的，具有自由商标的应用工具。

**（完整模式）**

*特别为编程语言优化*

应用程序运行在完整的环境中，并自动和计算机中已经安装的Visual Studio其他部件合并。

**问答**

**问：我如何才能使用****Visual Studio Shell?**

Visual Studio Shell将提供给我们的合作伙伴（VSIP）,包括beta 2版本，我们即将发布最终的版本。Visual Studio Shell是和Visual Studio SDK一起发布的。

**问：****Visual Studio Shell****费用如何？**

Visual Studio Shell将在Visual Studio SDK的正式版本中免费提供，而且建造和发布基于Visual Studio Shell的开发也是完全免费的。

**问：如何学习使用Visual Studio Shell？**

Visual Studio SDK包含了开发文档和示例，帮助你整合你的产品到Visual Studio Shell中，如果你是完美的VSIP，我们还可以专门为你解答一些棘手的技术问题。

**问：能够解释一些完整模式和独立模式吗？**

完整模式的应用程序是安装在Visual Studio中的，因此你能够充分利用现有的工具与之合作。独立模式的应用程序运行创建一个独立运行实例，与现有的Visual Studio完全独立工作，适合开发商提供自己的解决方案。完整模式看起来工作在Visual Studio下，而独立模式提供了更多的自定义选项。

**问：作为开发人员，我如何选择是使用完整模式还是独立模式****?**

首要问题是：你希望你的应用程序在用户的计算机中是工作在Visual Studio中，还是看起来和Visual Studio毫无关系。如果是第一个答案，你一概选择完整模式，否则你应该选择独立模式。
